# LucidGo v1.2.0

Changes since v1.1.0.

## New Features

### Compare Page

A new `/compare/` page lets users run side-by-side analysis across multiple games on a single shared board. Games are loaded from the Analyze page and must share the same SGF content so moves stay in sync.

* **Multi-lane overlay board** — top-move markers from each game are fan-offset on the board and color-coded using a 5-color high-contrast palette, so colliding positions remain distinguishable.
* **Compare legend** — color swatches and name chips displayed above the board identify each lane at a glance.
* **Compare sidebar** — color-coded tabs per lane give each game its own analysis config panel with Reset and Save (in-memory). A shared Controls and Win Rate section sits alongside. A **Run for all lanes** toggle triggers parallel analysis across every lane via `Promise.all`.
* **Mobile support** — the sidebar is rendered as a swipeable drawer on small screens.

### Comparison-Ready Indicator

A persistent indicator on the Analyze page shows when enough games are staged for comparison (as few as one game selected). Clicking it navigates directly to the Compare page.

### Auto-Save on Compare

Games are automatically saved when the user initiates a comparison from the Analyze page, so the Compare page always has a persisted session to load from.

### Game Name in Demo Page

The Demo page now surfaces the game name from SGF metadata, giving context when exploring a pre-loaded sample game.

## Refactoring

### Game Component Extraction

The `Game` component has been separated from `Analyze`, making the board, controls, and analysis panel independently reusable. The Demo page now uses this shared `Game` component.

### React Component Reusability

A broad reusability pass across the frontend consolidated duplicated board and control logic into shared components.

## Bug Fixes

* Fixed a max-depth crash when playing moves with the AI.
* Fixed incorrect state initialization in `Demo.tsx` that caused the sample game to load in a bad state.
* Fixed type mismatch errors in the frontend introduced during the compare work.
* Fixed an incorrect CSS width property in the comparison indicator.
